## Layer Architecture

The application consists of **5 layers**. Dependencies flow in one direction: upper → lower.

```
┌─────────────────────────────────────────────┐
│                  App Layer                   │  App lifecycle & UI entry point
│  AppDelegate / StatusBarController          │
├─────────────────────────────────────────────┤
│                 Core Layer                   │  Bridge to OS features
│  HotkeyManager                               │
├─────────────────────────────────────────────┤
│               Overlay Layer                  │  Window management & screen capture
│  OverlayWindow / OverlayWindowController    │
├─────────────────────────────────────────────┤
│                Draw Layer                    │  Drawing logic & rendering
│  DrawingCanvasView / ShapeRenderer / ...    │
├─────────────────────────────────────────────┤
│           Models + Utilities                 │  Data structures & extension methods
│  Stroke / DrawingState / Extensions         │
└─────────────────────────────────────────────┘
```

## Directory Roles

### `App/` — Application Lifecycle

| File | Role |
|---|---|
| `main.swift` | Explicit entry point. Calls `NSApplication.shared.run()`. (`@main` was not adopted because it failed to synthesize the entry point correctly) |
| `AppDelegate.swift` | Implements `NSApplicationDelegate`. Initializes the menu bar and hotkeys at launch, manages Draw mode toggling. Isolated with `@MainActor`. |
| `StatusBarController.swift` | Displays the menu bar icon via `NSStatusItem` and builds the menu. Provides actions for Draw launch, About, and Quit. |

**Design Notes:**
- `LSUIElement = YES` hides the Dock icon, making the app run as a menu bar resident
- Uses `NSApplicationDelegate` instead of SwiftUI's `@main App` (native AppKit event handling is required)

---

### `Core/` — OS Infrastructure

| File | Role |
|---|---|
| `HotkeyManager.swift` | Registers global hotkeys via Carbon `RegisterEventHotKey`. Detects ⌃2 press to launch Draw mode. As `@unchecked Sendable`, it safely fires to the main thread from the Carbon callback via `DispatchQueue.main.async`. No Accessibility permission required. |

**Design Notes:**
- Carbon `RegisterEventHotKey` was chosen. `CGEventTap` was rejected because it requires Accessibility permission, which gets invalidated on every rebuild
- Carbon callbacks are received via `EventHandlerUPP` and forwarded to the main thread with `DispatchQueue.main.async`

---

### `Overlay/` — Overlay Window Management

| File | Role |
|---|---|
| `OverlayWindow.swift` | `NSWindow` subclass. Borderless, transparent, always-on-top window. Receives keyboard events with `canBecomeKey = true`, displayed on all Spaces via `collectionBehavior`. |
| `OverlayWindowController.swift` | Manages the overlay lifecycle: creation, display, and teardown. Captures the screen using `ScreenCaptureKit` when Draw mode starts and passes it as a background to `DrawingCanvasView`. Isolated with `@MainActor`. |

**Design Notes:**
- `CGWindowListCreateImage` was deprecated in macOS 26, so `SCScreenshotManager.captureImage` is used instead
- Capture is performed asynchronously (`async/await`), with UI setup occurring after completion

### `Draw/` — Drawing Engine

The Draw layer is the core of the app. It is designed around a **3-layer compositing architecture**.

```
Layer stack during drawing (drawn bottom-to-top in draw(_:)):

  ┌──────────────────────────────┐
  │  activeFreehand (NSBezierPath) │  ← Freehand trail during drag
  ├──────────────────────────────┤
  │  previewLayer   (NSBezierPath) │  ← Shape during drag (line/rect/ellipse/arrow)
  ├──────────────────────────────┤
  │  finishedLayer  (CGImage)      │  ← Rasterized bitmap of all confirmed strokes
  └──────────────────────────────┘
```

| File | Role |
|---|---|
| `DrawingCanvasView.swift` | **Most critical file.** `NSView` subclass implementing 3-layer compositing. Handles drawing in `draw(_:)`, stroke management in `mouseDown/Dragged/Up`, and color changes, tool switching, and Undo in `keyDown/Up`. Exits Draw mode on Esc/right-click. ⌘C copies to clipboard, ⌘S saves to file. |
| `ShapeRenderer.swift` | Generates `NSBezierPath` for lines, rectangles, ellipses, and arrows. For arrows, the **drag start point is the tip** (ZoomIt-specific behavior). |
| `FreehandRenderer.swift` | Catmull-Rom spline interpolation for smoothing freehand paths. Converts raw mouse coordinates into smooth cubic Bézier curves. |
| `StrokeManager.swift` | Manages the Undo stack using `CGImage` snapshots. Maximum 30 levels. |
| `HighlighterRenderer.swift` | Rendering helper for semi-transparent highlighter strokes. Uses Multiply blend mode for realistic marker appearance. |
| `TextInputController.swift` | Text input mode activated by the T key. Places an `NSTextView` on the overlay, and on Escape confirmation, renders to an offscreen `CGBitmapContext` and burns it into `finishedLayer`. |

**Design Note — Shape switching via modifier keys:**

ZoomIt's defining characteristic is that shapes change only while modifier keys are held during a drag — not through toolbar or state switching.

```
mouseDown  → Record start point
mouseDragged →
  No modifier keys    → Freehand (point accumulation + Catmull-Rom)
  Shift held          → Line preview
  Control held        → Rectangle preview
  Tab held            → Ellipse preview (Tab is managed as a boolean via key events)
  Shift + Control     → Arrow preview
mouseUp    → Rasterize previewLayer into finishedLayer
```

Changing modifier keys mid-drag is immediately reflected. The `mouseDown` origin point is always preserved.

### `Models/` — Data Models

| File | Role |
|---|---|
| `Stroke.swift` | Struct representing a single stroke. Holds point array, color, width, shape type, and highlighter flag. |
| `DrawingState.swift` | Current drawing state (mutable). Manages active color, pen width, highlighter mode, text mode, Tab key state, and background mode (transparent/whiteboard/blackboard). Provides `currentShapeType(modifiers:)` method to derive `ShapeType` from modifier keys. |

**Auxiliary Types:**
- `ShapeType` — `freehand / line / rectangle / ellipse / arrow`
- `PenColor` — `red / green / blue / orange / yellow / pink` (with key character mapping)

## Data Flow

### Draw Mode Startup → Shutdown Flow

```
User: Presses ⌃2
    │
    ▼
HotkeyManager (CGEventTap callback)
    │ onDrawHotkey()
    ▼
AppDelegate.toggleDrawMode()
    │
    ▼
OverlayWindowController.showOverlay()
    │ ① Capture screen with ScreenCaptureKit
    │ ② Create OverlayWindow (fullscreen, transparent, always-on-top)
    │ ③ Place DrawingCanvasView (with captured image as background)
    ▼
DrawingCanvasView becomes key window
    │ Receives mouse/keyboard events
    │ Draws strokes → burns into finishedLayer
    ▼
User: Escape or right-click
    │
    ▼
DrawingCanvasView.onDismiss()
    │
    ▼
OverlayWindowController.dismiss()
    │ Destroys window
    ▼
AppDelegate.drawModeDidEnd()
    │ overlayController = nil
    ▼
Returns to normal state
```

### Stroke Confirmation Flow

```
mouseDown
    │ dragOrigin = current position
    │ freehandPoints = [current position]
    ▼
mouseDragged (branches by modifier keys)
    ├─ Freehand → append to freehandPoints → Catmull-Rom → activeFreehand
    └─ Shape    → dragOrigin to current position → ShapeRenderer → previewLayer
    │
    ▼ setNeedsDisplay(bounds) → draw(_:) composites 3 layers
    │
mouseUp
    │ ① strokeManager.pushUndoSnapshot(finishedLayer)  ← Save for Undo
    │ ② Composite finishedLayer + new stroke in CGBitmapContext
    │ ③ → Set new CGImage as finishedLayer
    │ ④ Set previewLayer / activeFreehand to nil
    ▼
    setNeedsDisplay(bounds)
```

## Technology Selection Rationale

### Swift + AppKit (SwiftUI Not Adopted)

| Aspect | AppKit | SwiftUI |
|---|---|---|
| Transparent overlay window | Direct control with `NSWindow` | Requires `NSViewRepresentable` hack |
| 3-layer compositing | Imperative control with `draw(_:)` + `CGContext` | `Canvas` is declarative, redraws everything each frame |
| Dragging with modifier keys | `mouseDragged` + `modifierFlags` | `DragGesture` cannot access modifier keys |
| Event handling | mouseDown/Dragged/Up/keyDown/flagsChanged | Insufficient granularity |

All Draw feature requirements map 1:1 to AppKit native APIs, hence its adoption.
Future settings screens can introduce SwiftUI via `NSHostingController`.

### Xcode Project (SPM Not Adopted)

`.app` bundle generation, `Info.plist`, code signing, and Entitlements management require native Xcode support.
Generated from `project.yml` (xcodegen), with the `.xcodeproj` itself tracked in Git.

## Required macOS Permissions

| Permission | Purpose | API |
|---|---|---|
| **Screen Recording** | Screen capture when Draw mode starts | `CGPreflightScreenCaptureAccess()` / `ScreenCaptureKit` |

**Accessibility permission is not required.** Global hotkeys use Carbon `RegisterEventHotKey`.

Behavior when permission is denied:
- Screen Recording denied → Drawing on black background (graceful degradation)
